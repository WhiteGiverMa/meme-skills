# meme-skills

中文互联网迷因（meme）知识库，以 open agent skill 格式维护。

## 设计意图

LLM 的训练数据具有滞后性，无法覆盖实时演变的互联网热梗。在个人助手场景中，AI 能否理解并回应用户提及的热梗，直接决定了对话的亲和力——懂梗的 AI 更像朋友，不懂梗的 AI 只是工具。

本项目将热梗知识标准化为 skill 格式，使任何兼容的 agent 框架都能按需加载。每个 skill 提供特定梗的起源、演变、用法、文化语境等完整信息，让 agent 在对话中准确理解并回应用户的梗文化表达。

```
    └── SKILL.md
```

## Skill 格式

每个 skill 是一个目录，内含 `SKILL.md`，结构如下：

```yaml
---
name: meme-<pinyin>
description: <梗名>（分类）知识库。当用户提及"<关键词1>""<关键词2>"或相关梗时触发。涵盖起源、演变、用法、衍生概念等完整信息。
---

# <梗名>（分类）

> **触发条件**：检测到关键词时的加载说明。

## 一、概述
## 二、起源
## 三、核心特征 / 含义
## 四、发展经过 / 传播时间线
## 五、衍生概念
## 六、文化分析
## 七、Agent 使用指南
## 参考资料
```

### description 字段规范

- 格式：`<梗名>（<分类>）知识库。当用户提及"<关键词>"或相关梗时触发。涵盖起源、演变……等完整信息。`
- 关键词选取最核心的 3-6 个，优先覆盖不同写法/别称
- 分类用 2-4 字概括（网络流行词、表情包迷因、空耳梗、无意义音效梗 等）

## 添加新 Skill 的流程

1. **搜索**：在百度百科、萌娘百科、弹幕秀、游侠手游、单词乎、B站梗百科、腾讯新闻等来源交叉验证
2. **核实**：至少 2 个独立来源确认关键事实（起源人物、时间线、核心含义）
3. **编写**：按上述模板创建 `meme-<pinyin>/SKILL.md`
4. **复核**：检查人物名、时间线、衍生概念的准确性

## 维护现有 Skill

- 梗的含义发生重大演变时更新对应板块
- 出现重要衍生事件（商业联动、官方玩梗等）时追加
- 发现事实错误立即修正
- 每次修改在 commit message 中注明变更原因

## 命名规范

- skill `name`：`meme-<拼音>`，全小写，连字符分隔（如 `meme-niyijiku`）
- 目录名与 `name` 一致
- 拼音不带声调，连续书写（嘉豪 = jiahao，我的刀盾 = wodedaodun）

## 安装

将 skill 目录复制到 agent 的 skills 目录即可：

```bash
# OpenCode / Sisyphus
cp -r meme-* ~/.agents/skills/
```
