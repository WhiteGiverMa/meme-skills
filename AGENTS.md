# meme-skills

中文互联网文化知识库，以 agent skill 格式维护。覆盖两大领域：**热梗知识**（meme-{xx}）与**游戏知识**（game-{xx}）。

## 设计意图

LLM 的训练数据具有滞后性，无法覆盖实时演变的互联网热梗与持续更新的游戏知识。在个人助手场景中，AI 能否理解并回应用户提及的梗文化和游戏话题，直接决定了对话的亲和力——懂梗的 AI 更像朋友，不懂梗的 AI 只是工具；了解用户玩的游戏的 AI 能聊到一起，不了解的只能泛泛而谈。

本项目将这些知识标准化为 skill 格式，使任何兼容的 agent 框架都能按需加载：

- **meme-{xx}**：热梗知识。每个 skill 提供特定梗的起源、演变、用法、文化语境等完整信息，让 agent 在对话中准确理解并回应用户的梗文化表达。
- **game-{xx}**：游戏知识。覆盖热门新游戏与经典小众冷门游戏，提供游戏机制、开发历史、社区文化、玩家梗等信息，让 agent 能与玩家用户进行有深度的游戏话题交流。

## 人文精神

热梗不仅是笑话。每一个在网络中生长、变异、沉淀下来的梗，都是一段微缩的社会史——它记录了某个时刻一群人的情绪、一段关系的变迁、一种文化的自我表达。嘉豪的走红折射了当代年轻人对亲密关系的渴望与戏谑；比比拉布的无意义音效背后是互联网对语言边界的解构与重建。

本项目不将梗视为"网络垃圾"或"低级趣味"，不将游戏视为"消磨时间的娱乐"。相反，我们认为：**理解一个时代的梗，就是理解这个时代的人；理解一个人玩的游戏，就是理解TA的审美与精神世界。** 每一个 skill 不仅是对一个流行词的百科式记录，更是对一种社会情绪、一段文化记忆、一种审美偏好的尊重与保存。

在编写和维护 skill 时，请始终秉持以下态度：
- **准确而非猎奇**：还原梗的真实起源和语境，不为了趣味性而编造或夸大
- **理解而非嘲讽**：梗背后是真实的人和真实的情感，不做居高临下的评判
- **完整而非碎片**：提供足够的文化语境，让 AI agent 不仅"知道"这个梗，更"理解"它为什么好笑、为什么重要
- **尊重而非消费**：涉及特定群体、地域、性别的梗时，保持文化敏感度，不将其简化为刻板印象

> 一个真正称职的个人 AI 助手，不该只是个工具。它应该懂你的梗，接得住你的玩笑。而要做到这一点——首先，写 skill 的人要懂这些梗背后的人。

## Skill 格式

每个 skill 是一个目录，内含 `SKILL.md`，结构如下：

```yaml
---
name: meme-<pinyin>
description: <梗名>（分类）知识库。当用户提及"<关键词1>""<关键词2>"或相关梗时触发。涵盖起源、演变、用法、衍生概念等完整信息。
---

# <梗名>（分类）

> **触发条件**：检测到关键词时的加载说明。

## 一、概述
## 二、起源
## 三、核心特征 / 含义
## 四、发展经过 / 传播时间线
## 五、衍生概念
## 六、文化分析
## 七、Agent 使用指南
## 参考资料
```

### description 字段规范

- 格式：`<梗名>（<分类>）知识库。当用户提及"<关键词>"或相关梗时触发。涵盖起源、演变……等完整信息。`
- 关键词选取最核心的 3-6 个，优先覆盖不同写法/别称
- 分类用 2-4 字概括（网络流行词、表情包迷因、空耳梗、无意义音效梗 等）

## 添加新 Skill 的流程

> **重要**：新 skill 一律先在 `incoming/` 目录中起草，经复核确认无误后再移入 `skills/` 正式目录。

### 热梗 Skill

1. **搜索**：在百度百科、萌娘百科、弹幕秀、游侠手游、单词乎、B站梗百科、腾讯新闻等来源交叉验证。若搜索结果缺少信息，应回复并暂停工作
2. **核实**：至少 2 个独立来源确认关键事实（起源人物、时间线、核心含义）
3. **起草**：按上述模板在 `incoming/` 中创建草稿 `incoming/meme-<pinyin>/SKILL.md`
4. **复核**：检查人物名、时间线、衍生概念的准确性
5. **移入正式目录**：复核通过后，将 `incoming/meme-<pinyin>/` 移入 `skills/`，并更新 README.md 与 AGENTS.md 中的目录索引

### 游戏 Skill

游戏 skill 由人工编写把关，不遵循自动模板流程。编写时先在 `incoming/` 中起草，完成后移入 `skills/`。

## 维护现有 Skill

- 梗的含义发生重大演变时更新对应板块
- 出现重要衍生事件（商业联动、官方玩梗等）时追加
- 发现事实错误立即修正
- 每次修改在 commit message 中注明变更原因

## 命名规范

- meme skill `name`：`meme-<拼音>`，全小写，连字符分隔（如 `meme-niyijiku`）
- game skill `name`：`game-<英文名>`，全小写，连字符分隔（如 `game-slay-the-spire-2`）
- 目录名与 `name` 一致
- meme：拼音不带声调，连续书写（嘉豪 = jiahao，我的刀盾 = wodedaodun）
- game：使用英文原名，空格替换为连字符（Subnautica = subnautica，Slay the Spire 2 = slay-the-spire-2）

## 安装

将 skill 目录复制到 agent 的 skills 目录即可：

```bash
# OpenCode / Sisyphus
cp -r skills/meme-* ~/.agents/skills/
cp -r skills/game-* ~/.agents/skills/
```
